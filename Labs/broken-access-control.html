Broken Access Control Labs

   
     

Access control decides what authenticated users can actually do.

     
           
  • Authentication — who you are
    •        
  • Session Management — tracking your requests
    •        
  • Access Control — what you're allowed to touch
    •      
     

Broken access control = one of the most dangerous and common web vulnerabilities.

   
   

Vertical Privilege Escalation

   
     

Moving from low privilege → high privilege (e.g. regular user → admin).

     

Example: non-admin deletes accounts via /admin panel.

   
   

Unprotected Functionality

   
     

Admin-only pages hidden only by obscure URLs → security by obscurity (fails hard).

     

Lab: Unprotected Admin Functionality

           
           
  • Intercepted traffic → spotted My Account flow
    •        
  • Brute-forced /admin, /admin-panel, etc.
    •        
  • Found working endpoint → 200 OK → deleted users → lab solved
    •      
           

Lab: Unprotected Admin Functionality with Unpredictable URL

     

Key: admin URL leaked in JavaScript on product page.

           

→ direct request → 200 → user deletion → solved.

   
   

Parameter-based Access Control

   
     

Role/rights stored in manipulable locations:

     
           
  • hidden input
    •        
  • cookie
    •        
  • query param (?admin=true, ?role=1)
    •      
     

Lab: User Role Controlled by Request Parameter

     

Steps:

     
           
  • Found admin=false in login / cookie
    •        
  • Flipped to true in multiple requests
    •        
  • Manipulated cookie value → accessed admin panel → deleted users
    •      
   
   

Horizontal Privilege Escalation

   
     

Accessing another user's resources at the same privilege level.

     

Example: change ?id=123 → ?id=456 (or GUID)

     

Lab: User ID Controlled by Request Parameter (Unpredictable IDs)

     
           
  • Found Carlos' GUID via article author link / profile click
    •        
  • Swapped own account ID → accessed Carlos' data
    •        
  • Leaked API key: io8KpnnXtpJE6hm4yJvdrIu7n3H1dqpc
    •      
         
   

Prevention

   
     
           
  • Never trust obscurity / hidden URLs
    •        
  • Deny by default
    •        
  • Centralized, enforced access control layer
    •        
  • Declare permissions explicitly in code
    •        
  • Audit + test access paths religiously
